# Git 日常指令小抄

用「雲端相簿」的概念來記憶：GitHub 就是雲端相簿，你的電腦是你手上的相機。

## 初次使用 / 換電腦

```bash
git clone 網址
```
把雲端的東西載下來（把整本相簿下載到這台電腦）。

```bash
git pull
```
換到別台電腦要開始改之前、或之後每次要繼續改之前，先「同步最新版本」（確認相簿裡沒有別人新增的照片，避免衝突）。

## 日常使用的固定流程

三個指令，每次存檔都照這個順序做：

```bash
git add .
```
把要存的東西挑出來（從一堆照片裡，先挑出這次要存檔的幾張）。

```bash
git commit -m "這次改了什麼（用文字描述）"
```
拍照存檔，附上這次的備註（例如「修正登入頁面的錯字」）。

```bash
git push
```
把照片上傳到雲端相簿（同步到 GitHub，其他人或其他電腦才看得到）。

## 記憶口訣

**clone → pull → (add → commit → push) 重複循環**

- `clone`：只有第一次要做
- `pull`：每次開始改之前，先確認自己手上的是最新版
- `add / commit / push`：每次改完，存檔 + 上傳，三步驟一定要照順序
